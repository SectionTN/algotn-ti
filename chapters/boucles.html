↻ Les boucles

Répéter sans se répéter

Imagine devoir afficher « Bonjour » 100 fois. Tu ne vas pas écrire 100 lignes ! Une boucle dit « refais ça tant qu'il le faut ». C'est le vrai super-pouvoir de l'algorithmique.

1 Les 3 boucles

BoucleQuand l'utiliserTest
POURNombre de tours connu à l'avanceCompteur géré automatiquement
TANT QUENombre de tours inconnu ; test avantPeut ne jamais s'exécuter
RÉPÉTERTest après ; s'exécute au moins une foisIdéal pour les saisies à valider

2 Syntaxe

boucles.algo
🔁
POUR = TANT QUE déguisé. Pour i De 1 À n revient à : i←1 ; tant que i≤n { … ; i←i+1 }. Le POUR gère juste le compteur pour toi.

3 Loop Visualizer

Choisis un type de boucle, fixe n, et regarde le compteur grimper, la condition se tester et la somme s'accumuler à chaque tour.

4 Le motif « accumulateur »

Le schéma le plus utile : une variable qu'on met à jour à chaque tour (somme, produit, compteur, max…). On l'initialise avant la boucle.

somme.algo

5 Erreurs fréquentes

  • Boucle infinie : la condition d'arrêt n'est jamais atteinte (oubli d'incrémenter).
  • Oublier d'initialiser l'accumulateur avant la boucle.
  • Erreur de bornes (« décalage d'un » : commencer à 0 ou 1, finir à n ou n−1).
  • Choisir TANT QUE alors qu'il faut RÉPÉTER (saisie à exécuter au moins une fois).
🎯
Astuce examen. « Lire jusqu'à une valeur correcte » ⟹ RÉPÉTER (test après). « Parcourir un tableau de n cases » ⟹ POUR. « Tant qu'il reste un nœud » ⟹ TANT QUE.

8 Mini-résumé

  • POUR = nombre de tours connu ; TANT QUE = test avant ; RÉPÉTER = test après (≥1 tour).
  • Accumulateur : initialiser AVANT, mettre à jour DANS la boucle.
  • Boucle infinie = condition d'arrêt jamais atteinte.
  • Attention aux bornes (0..n−1 vs 1..n).
