𝒪 La complexité

Pourquoi un algorithme est rapide… ou lent

Deux algorithmes peuvent donner le même résultat, mais l'un en une seconde et l'autre en une heure. La complexité mesure comment le temps de calcul grandit quand les données grandissent. C'est ce qui sépare un programme utilisable d'un programme inutilisable.

1 L'idée : comment ça « scale » ?

📈
On ne compte pas les secondes. Le temps dépend de la machine. On compte le nombre d'opérations en fonction de la taille des données n, et surtout comment ce nombre grandit quand n explose. C'est la notation « grand O ».

2 La notation O()

On garde seulement le terme qui domine quand n est grand, sans les constantes. Exemple : 3n + 5 → O(n) ; 2n² + 10n → O(n²).

NotationNomExemple typique
O(1)ConstantAccès T[i], push/pop d'une pile
O(log n)LogarithmiqueRecherche dichotomique
O(n)LinéaireParcours d'un tableau/liste, recherche séquentielle
O(n log n)Quasi-linéaireTris rapides (fusion, rapide)
O(n²)QuadratiqueTris simples, double boucle imbriquée

3 Graphique comparatif

Fais glisser n et regarde le nombre d'opérations. À n=64, O(log n) ≈ 6 alors que O(n²) ≈ 4096. La différence devient gigantesque !

4 Lire la complexité d'un code

  • Une instruction simple (affectation, comparaison) → O(1).
  • Une boucle de 0 à n → O(n).
  • Deux boucles imbriquées sur n → O(n²).
  • On divise le problème par 2 à chaque étape → O(log n).

6 Erreurs fréquentes

  • Garder les constantes (O(2n) s'écrit O(n)).
  • Croire qu'« ça marche sur mon petit exemple » suffit : tester avec un grand n.
  • Confondre une boucle qui divise par 2 (log n) et une qui décrémente de 1 (n).
  • Oublier que la dichotomie exige un tableau trié.
🎯
Astuce examen. Pour donner la complexité : compte les boucles. Une boucle sur n = O(n). Deux imbriquées = O(n²). Division par moitié = O(log n). Garde le terme dominant, oublie les constantes.

7 QCM

8 Mini-résumé

  • La complexité mesure la croissance du nombre d'opérations selon n.
  • Ordre : O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ).
  • Boucle simple = O(n) ; imbriquée = O(n²) ; division par 2 = O(log n).
  • On garde le terme dominant, sans les constantes.
