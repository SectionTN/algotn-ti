Les fondations : DÉBUT, variables, FIN

Un algorithme, c'est une recette : une suite d'instructions précises pour passer de « ce qu'on a » à « ce qu'on veut ». Avant les boucles et les pointeurs, il faut maîtriser ces briques de base. Elles reviennent dans absolument tout.

1 Le squelette d'un algorithme

Tout algorithme a la même charpente : un nom, une zone de déclarations, puis le corps entre DÉBUT et FIN.

squelette.algo
🤔
Pourquoi déclarer ? Déclarer une variable, c'est réserver une case mémoire et lui donner un nom + un type. Le type dit quelle sorte de valeur la case accepte (entier, réel, caractère, chaîne, booléen).

2 Variables, constantes & affectation

⚠️
Le sens de la flèche compte x ← y copie y DANS x (x change, y inchangé). x ← x + 1 se lit « la nouvelle valeur de x = ancienne + 1 ». Ce n'est pas une égalité mathématique !

3 Entrées / Sorties : Lire & Ecrire

  • Lire(x) → demande une valeur à l'utilisateur et la range dans x (entrée).
  • Ecrire(x) → affiche la valeur de x à l'écran (sortie).

4 Simulateur : exécute pas à pas

Change A et B, lance, et regarde la mémoire et la sortie évoluer ligne par ligne. C'est exactement ce que fait la machine.

5 Erreurs fréquentes

  • Utiliser une variable avant de lui donner une valeur (contenu indéterminé).
  • Confondre = (test d'égalité) et ← (affectation).
  • Mauvais type : ranger un réel dans un entier perd la partie décimale.
  • Oublier de Lire avant d'utiliser une donnée saisie.
🎯
Astuce examen. Pour « échanger deux variables », il faut une 3ᵉ variable d'aide : aux←a ; a←b ; b←aux. Sans aux, on écrase une valeur.

8 Mini-résumé

  • Charpente : nom → déclarations → DÉBUT … FIN.
  • Variable = case nommée typée ; constante = valeur fixe.
  • ← = affectation (pas une égalité).
  • Lire = entrée, Ecrire = sortie.
