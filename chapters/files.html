▤ Les files

La file : premier arrivé, premier servi

La file d'attente à la boulangerie : le premier arrivé est servi en premier, on entre par la fin, on sort par le début. C'est une file (queue). Principe : FIFO (First In, First Out). L'exact opposé de la pile.

1 Le principe FIFO

🥖
Analogie : la file d'attente. On rejoint la queue (derrière), on est servi depuis la tête (devant). Pas de passe-droit : l'ordre d'arrivée est respecté.

Deux opérations principales :

  • Enfiler (enqueue) : ajouter un élément en queue.
  • Défiler (dequeue) : retirer l'élément de tête.

2 Queue Visualizer

Enfile (ça s'ajoute à droite, en queue) et défile (ça part par la gauche, en tête). L'ordre d'arrivée est toujours respecté.

4 Applications classiques

  • Files d'attente : impression, tâches, serveurs.
  • Gestion de commandes/colis dans l'ordre d'arrivée.
  • Parcours en largeur d'un graphe/arbre (BFS).
  • Tampons (buffers) de données.
🎯
Astuce examen. « Dans l'ordre d'arrivée », « premier arrivé premier servi », « file d'attente » ⟹ file (FIFO). « Dernier d'abord », « annuler », « inverser » ⟹ pile (LIFO).

5 Erreurs fréquentes

  • Défiler une file vide.
  • Confondre tête (sortie) et queue (entrée).
  • Confondre FIFO (file) et LIFO (pile).
