▤ Les enregistrements

Regrouper ce qui va ensemble

Un étudiant a un nom (chaîne), un âge (entier) et une moyenne (réel). Trois infos de types différents, mais qui décrivent UNE même chose. L'enregistrement (ou « structure ») les réunit sous un seul nom. C'est la brique des bases de données et des listes chaînées.

3 Tableau d'enregistrements (le combo gagnant)

Gérer 100 étudiants ? Un tableau d'enregistrements : chaque case est un étudiant complet. C'est la structure typique des sujets de gestion (produits, véhicules, articles…).

tableau_struct.algo
🔗
Pont vers les listes chaînées. Un nœud de liste chaînée EST un enregistrement : un champ donnée + un champ suivant (pointeur). Maîtriser les enregistrements, c'est déjà préparer les listes.

4 Erreurs fréquentes

  • Oublier le point : écrire e au lieu de e.nom.
  • Confondre le type (Etudiant) et la variable (e).
  • Pour un tableau d'enregistrements : oublier l'indice et le champ (classe[i].nom).
  • Mauvais type de champ (mettre une chaîne dans un champ entier).
🎯
Astuce examen. Sur un sujet de gestion : ① définis le type Enregistrement, ② déclare un tableau de ce type, ③ écris une procédure Saisir(var T, n) et une Afficher(T, n), ④ ajoute les traitements demandés. Structure quasi systématique.
